палитра
на мониторах второго уровня действует кастомная палитра для улутшения картинки
однако в файле /etc/pallete/gpu/depth4.cfg ее можно переопределить
там же можно определить палитру для depth8 в файле /etc/pallete/gpu/depth8.cfg
при редектировнии палитры depth4 сдедует так же указать эти цвета в качестве основных системных переместив файл в
/etc/pallete/colorPic/depth4.cfg а для того чтобы цвета были изменены в программах с интеграцияй и на depth8 его можно скопировать и в
/etc/pallete/colorPic/depth8.cfg однако все цвета бедет идти через палитру только в режиме depth4
для того чтобы узнать режим depth необходимо ввести depth
для того чтобы устоновить depth и режим 1 4 или 8
если режим не поддерживаеться это будет написано

устоновка программ
вы можете воспользоваться market
если хотите устоновить программу с дискеты
используйте устоновшик install он тут кастосный
он видит только дискеты для устоновки и ссылаеться на оригинальный устоновшик
однако генерирует файл позваляюший быстро удалить устоновленую программ
утилитой uninstall
утилита uninstall позваляет удалять программы не только с root диска но и с стороньних
и даже подключенного к зарядке планшета
программа install тоже может ставить на все диски включая tmpfs
тоесть вы можете спакойно поставить loot дискету на планшет с компа
а потом с компа же ее и удалить
что очень удобно
программа install сканирует всю файловыю систему куда производиться устоновки перед и после устоновки
все новые файлы включает в удалятор, именно по этому нельзя удалять через uninstall программу которая была
устоновлена рания даже если вы устоновите ее вновь и она появиться в uninstall она всеравно не будет удалена
тогда ее придеться удалить вручьную
токже программа install имеет ключ -a который позваляет производить
устоновку со всех дисков на все игнорируя все правила этих дисков и самой ос(ИСПОЛЬЗОВАТЬ КРАЙНЕ ОСТОРОЖНО)
обратите внимания что в обычьном режиме один диск не может быть и loot и target
он будет target если он не loot не readonly и у него нет ключа nonTarget

завершения работы
крайне желательно завершать работу только програмно
такое завершения работы предварительно генерирует event shutdown
а зетем ждет сикунду для того чтобы все узлы системы и стороньнего софта
коректно завершили свою работу
любая програма может отлажить завернения работы на неопределенный скор
для этого ей требуеться сгенирировать event postponeShutdown а первым аргументом передать любое уникальное значения
для того чтобы позволить закончить завершения работы необходимо сгенерировать event allowShutdown и передать туда тоже случайное значения
только когда все программы откладывавшие завернения работы передадут allowShutdown только тогда произойдет завершения работы

recovery режим
режим который может помочь если с системой траблы
загрузиться в него можно используя такие биосы как open loader/smart efi/micro bios
в нем настройки системы стандартные
не рекомендуеться менять настройки в этом режиме так как это повлечет за собой оброс настроик в стандарном режиме
в этом режиме нет контроля заряда и многого другого присушему обычьной загрузке
в нем игнорируються настройки дисков
ипользовать только в случаи сбоя

запуск программ
вы можете запустить программу с внешнего диск если она лежит на нем в папке /bin /usr/bin /home/bin
также со стороньних дисков будут подгружаться библиотеки из /lib /usr/lib /home/lib
опять таки приведу пример с планшетом если вы на него скачаете прогрмуу из market то подключив
планшет к компу через зарядку вы сможете запустить эту программу

программа programs
это менаджер программ
чтобы вывести список ВСЕ программ напишите programs allList(игнорит повторения)

чтобы вывести список программ зарегистрировавшихся в programs напишите programs list
также вы можете успровлять такими программами programs command name ...

чтобы вывести список программ пользователя введите programs userList(игнорит повторения)
выведет список всех программ которые пренадлежат пользователю
тоесть тех чего имени нет в /bin тоесть если программа лежин на стороньнем диске в bin то она считаеться программой пользователя
а если же программа с таким же именем лежит в bin на системном диске то та на стороньнем уже не считаеться пользовательской

контроль питания
система резирвирует 1/8 часть энергии под коретное завершения работы и все программы также не видят этот кусок
когда энергии становить меньше 20%(без учета резервированого фрагмента) раз в 25 сек будет появляться окно говоряшия о малом заряде батареи
когда энергии будет 0%(без учета резервированого фрагмента) появиться логорип перечеркнутой батареи и произойдет отключения

настройки дисков
вы можете обознать поведения дисков
например сделать жеский диск который будет определяться install как loot дискета или наоборот невкакую не определяться
создав на данном диске файл .perms.cfg(он ну будет работать в recovery mod)
там необходимо поместить обычьную lua табличьку
там могут быть ключи:
nonTarget(диск не будет определен как тот куда может быть произведена устоновка через install)
doNotMount(не монтировать в mnt(диск всеравно будет монтироваться в /free/allMounts))
noLoot(не лут дискета даже если это дискета)
isLoot(будет определяться как лут дискета даже если это жеский диск)
